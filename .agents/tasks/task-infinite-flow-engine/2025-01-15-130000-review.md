# Infinite Flow Engine v2.0 — fix pass (v2)

The fix commit addresses all seven issues raised in the first review: `generate_infinite_flow()` is now wired into Cell 9's `USE_SEGMENT_EXTENSION` branch, `blend_into_latent` performs actual temporal-decay blending, `_clean_output` has ~30 regex passes, content-tier detection and pacing logic are present in `generate()`, VRAMManager settings are consumed by `generate_pro()`, `@vram_guard` decorates `generate_pro()`, FlowState gets populated with segment_paths/prompts_used/seeds_used/total_frames, the `PYTORCH_CUDA_ALLOC_CONF` duplicate is replaced with a comment, quality keywords are appended directly to `user_input` in scene dicts, and Cell 1 pip commands are clearly annotated as Colab-only.

**Watch for:** FlowState is created internally and never returned to the caller, making the tracking data inaccessible from Cell 9 (confirmed). `_build_negative_prompt` still lacks the content-tier-aware negatives (`_NEG_EXPLICIT`, `_NEG_WIDE`, `_NEG_MULTI`) present in the original `LTX2EasyPromptLD.py` (confirmed). `PersistentLatentSeed.set_reference` generates a 1-element noise tensor for non-Tensor inputs, which will crash `blend_into_latent`'s interpolation path (confirmed, pre-existing but now reachable via the new blend logic).

**Verdict**: NEEDS_CHANGES

## High-level view

Cell 9 dispatches to `generate_infinite_flow()`, which creates an `InfiniteFlowConfig` from globals and delegates to `generate_extended_video()`. The post-generation FlowState update now scans the segment cache directory for actual file counts, records prompts and seeds, and computes overlap-adjusted total frames — but this state object is created locally and never surfaced to the caller, so the tracking is dead on arrival.

`_clean_output` went from 10 patterns to approximately 30, covering instruction labels, self-eval patterns, model loop/panic, filler-char spam, compliance checklists, timestamp annotations, and screenplay directions. The content-tier detection adds explicit/sensual/neutral instruction injection, and pacing enforcement appends token-count guidance. However, `_build_negative_prompt` was not updated to include the explicit-content, wide-shot, or multi-character negative categories from the source.

VRAMManager integration caps frames and forces tiled VAE on low-VRAM GPUs but does not override resolution or model selection — those remain manual globals.

1. **FlowState inaccessible from Cell 9** — `generate_infinite_flow()` creates `flow_state` internally when `None` is passed (the default), populates it, then returns only the video path. The populated FlowState is garbage-collected. Either return a tuple `(result, flow_state)` or store the state on a module-level variable.

The inline version only handles indoor/outdoor, night/day, and portrait keywords. Explicit content and multi-character scenes will get weaker negative guidance.
3. **`set_reference` with non-Tensor input produces unblendable noise** — When `reference_image` is not a `torch.Tensor`, `img_data` is `torch.zeros(1)` and `_noise_tensor` becomes a 1-element tensor. The new `blend_into_latent` then tries `F.interpolate` on this, which requires at least 4D input. This path will raise a RuntimeError if a non-Tensor reference is ever passed and blending is attempted.

## FlowState created and discarded

```python
def generate_infinite_flow(..., flow_state=None, ...):
    if flow_state is None:
        flow_state = FlowState()
    ...
    if result:
        flow_state.segment_index = config.max_segments

        flow_state.prompts_used.append(...)
        flow_state.seeds_used.append(...)
    return result  # flow_state is lost
```

Cell 9 calls `output = generate_infinite_flow(...)` without passing `flow_state`. The Cell 11 usage instructions tell users that "Infinite Flow mode" provides "FlowState tracking", but that state is unreachable without explicitly constructing and passing a `FlowState` object — which the default Cell 9 call doesn't do.

## `_build_negative_prompt` content gaps

The original (`LTX2EasyPromptLD.py` lines 32-54) scans for explicit terms, wide-shot terms, and multi-character terms:

```python
_NEG_EXPLICIT = "censored, mosaic, pixelated, black bar, blurred genitals"
_NEG_WIDE     = "close-up, portrait crop, tight frame"
_NEG_MULTI    = "merged bodies, fused figures, incorrect number of people"
```

These three categories are absent from the inline `_build_negative_prompt`. Since content-tier detection was added to `generate()`, the negative prompt builder should match: explicit content needs "censored, mosaic" negatives, multi-character scenes need "merged bodies" negatives, and wide shots need "close-up crop" negatives.

## `PersistentLatentSeed` dimension mismatch path

`set_reference` when `reference_image` is not a Tensor:

```python
img_data = torch.zeros(1)  # shape: (1,)
self._noise_tensor = torch.randn_like(img_data, generator=generator)  # shape: (1,)
```

`blend_into_latent` then attempts:

```python
noise = self._noise_tensor  # shape: (1,)
if noise.ndim == 3: noise = noise.unsqueeze(0)  # skipped, ndim==1
noise_resized = F.interpolate(noise[:1, :latent_tensor.shape[1]], ...)
```

A 1D tensor cannot be indexed with two-dimensional slicing — this raises an IndexError. Fix by either rejecting non-Tensor references in `set_reference` (early return with `_noise_tensor = None`) or generating noise with spatial dimensions matching a default size.
