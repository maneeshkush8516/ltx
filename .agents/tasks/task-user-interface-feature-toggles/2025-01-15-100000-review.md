# Quick Setup Dashboard and Feature Toggles for LTX_PRO

A two-part addition to the Colab notebook: Cell 3.5 introduces a preset-driven configuration dashboard (four GPU-tier presets, resolution shortcuts, generation mode selector, difficulty level), and Cell 3.6 adds ten `ENABLE_*` boolean toggles that disable major subsystems. The second commit wires those toggles into `generate_pro()` so disabled features actually skip their pipeline phases — video generation master switch, upscaling (Phase 7), audio decode, character system, and post-processing all gate on their respective toggles.

**Watch for:** Two toggles (`ENABLE_QUALITY_CHECKS`, `ENABLE_SCENE_CONTINUITY`) are defined and assignable but never read in `generate_pro()`, meaning they have no runtime effect (confirmed). The resolution preset always overrides the generation preset's width/height, making the "Cinema Quality" preset's 1024x576 dead on arrival if `RESOLUTION_PRESET` stays at its default "Landscape 16:9" (confirmed). The `vae_audio` deletion after the audio decode block will crash with `NameError` if audio was never loaded earlier in the pipeline and the toggle skipped it (likely).

## High-level view

The preset system in Cell 3.5 uses a two-layer override model: generation preset sets all parameters, then resolution preset clobbers width/height. The ordering means the generation preset's carefully chosen resolution is silently discarded whenever the resolution preset isn't "Custom". This is a UX trap — "Cinema Quality" advertises 1024x576 but the user gets 768x512 unless they separately switch resolution to "Custom".

The ten feature toggles are straightforward booleans with Colab `@param` annotations. Two utility functions (`disable_all_extras`, `enable_recommended`) batch-toggle them by GPU tier.

Integration into `generate_pro()` is partial. The toggles that map to existing variables (`ENABLE_LLM_PROMPT_EXPANSION` -> `_bypass`, `ENABLE_VISION_ANALYSIS` -> `_use_vision`) work via override-on-read. The phase-skip toggles (`_enable_upscaling`, `_enable_audio`, `_enable_character`, `_enable_post_processing`) gate code blocks directly. But `_enable_quality_checks` is assigned and never referenced again, and `ENABLE_SCENE_CONTINUITY` isn't even resolved into a local variable — both are inert.

## Preset override ordering in Cell 3.5

`apply_preset()` applies the generation preset first (setting WIDTH, HEIGHT, FRAMES, etc.), then unconditionally applies the resolution preset on top. The resolution preset defaults to "Landscape 16:9" (768x512):

```python
# User selects "Cinema Quality" expecting 1024x576
GENERATION_PRESET = "Cinema Quality"
RESOLUTION_PRESET = "Landscape 16:9"  # default
apply_preset()
# Result: WIDTH=768, HEIGHT=512 — not what "Cinema Quality" promised
```

Only "Cinema Quality" (1024x576) and "Quick Draft" (512x512) are silently overridden — the other presets happen to share 768x512 with "Landscape 16:9" so the conflict is invisible. The fix is either to set `RESOLUTION_PRESET = "Custom"` inside `apply_preset()` when a generation preset is selected, or to skip the resolution override when the generation preset's resolution differs from the default.

## Inert toggles: quality checks and scene continuity

`ENABLE_QUALITY_CHECKS` is described as "Maps to USE_QUALITY_GATE" in the comment, but the integration commit never bridges them. Inside `generate_pro()`, `_enable_quality_checks` is assigned on line 5902 and never referenced again. The existing `_use_quality_gate` variable (resolved from `USE_QUALITY_GATE` earlier) continues to control the quality gate logic independently. A user toggling `ENABLE_QUALITY_CHECKS = False` gets no effect.

`ENABLE_SCENE_CONTINUITY` isn't even resolved into a local `_enable_*` variable inside `generate_pro()`. The comment says it "Maps to USE_SCENE_CONTINUITY" but no such mapping exists in the integration code.

## Upscale-skip path and latent format mismatch

When `_enable_upscaling` is False, the code takes this branch:

```python
else:
    print("\n   [SKIPPED] Phase 7 - Upscaling disabled via ENABLE_UPSCALING=False")
    print("   Decoding directly from Pass 1 output.")
    p2_denoised = p1_av
    ltxvsep = NODE_CLASS_MAPPINGS["LTXVSeparateAVLatent"]()
    del unet
    aggressive_cleanup("Pass 2 skipped - UNet freed")
```

`LTXVSeparateAVLatent` is instantiated but never executed. Downstream in Phase 8, the decode path expects separated video and audio latents (`vid_lat_fin` and `aud_lat_fin`). Assigning `p2_denoised = p1_av` without actually calling the separator means the combined AV tensor flows into decode unchanged. This is the most architecturally concerning skip path because it changes the tensor format reaching Phase 8.

## `vae_audio` lifecycle when audio is skipped

After the audio decode block, `del vae_audio` runs unconditionally regardless of `_enable_audio`. Currently the audio VAE loads earlier in the pipeline regardless of the toggle, so this works. But the unconditional `del` becomes a `NameError` crash the moment someone adds early-exit audio loading gated on `_enable_audio`. A `if vae_audio is not None: del vae_audio` guard would make this robust against future toggle expansion.

## Feature toggles bypass the parameter-override pattern

Every other configuration in `generate_pro()` follows `_var = param if param is not None else GLOBAL`. The feature toggles break this:

```python
_enable_video = ENABLE_VIDEO_GENERATION
_enable_upscaling = ENABLE_UPSCALING
```

No corresponding function parameters exist. A caller who wants to generate one clip with upscaling disabled and another with it enabled must mutate globals between calls. In a Colab notebook this is tolerable, but it prevents programmatic batch runs with different toggle configurations without global side effects.
