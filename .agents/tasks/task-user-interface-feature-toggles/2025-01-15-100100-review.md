## High-level view

The five v1 issues are addressed. The resolution override fix creates a new tradeoff: users cannot combine a generation preset's timing/LLM/VAE settings with a separate resolution choice, because `apply_preset()` always forces `RESOLUTION_PRESET = "Custom"`. A "None" or "Manual" entry in the generation preset dropdown would restore that flexibility, but this is a minor UX polish item rather than a blocking concern.

Scene continuity uses a module-level sync at Cell 6 execution rather than a runtime override inside `generate_pro()` — it works given Colab's linear execution order but is the only toggle with cell-execution-time binding rather than call-time binding. The asymmetry exists because `USE_SCENE_CONTINUITY` is consumed by `run_storyboard()`, not `generate_pro()`.

## Resolution preset override — from one problem to another

The implementation (`RESOLUTION_PRESET = "Custom"` inside `apply_preset()`) eliminates the resolution preset as an independent control surface. The `RESOLUTION_PRESET` Colab widget still appears in the UI, still accepts user input, and still has five visible options — but its value is immediately discarded:

```python
# User sets both widgets:
GENERATION_PRESET = "Standard"        # wants 121 frames, 3B LLM, etc.
RESOLUTION_PRESET = "Portrait 9:16"   # wants 576x1024

apply_preset()
# RESOLUTION_PRESET is now "Custom" (forced by generation preset)
# WIDTH=768, HEIGHT=512 (from "Standard" preset)
# User's portrait choice is gone
```

The dropdown's presence without effect is worse than removing it, because it implies the user has control they don't have. A minimal fix: add `"Manual"` to the `GENERATION_PRESET` `@param` list, with no corresponding entry in `_PRESET_DEFINITIONS`. When selected, `preset` would be `None`, skipping the generation override entirely and letting `RESOLUTION_PRESET` drive width/height.

## Scene continuity sync mechanism

The sync lives at module level in Cell 6:

```python
USE_SCENE_CONTINUITY = True  # @param
if not ENABLE_SCENE_CONTINUITY:
    USE_SCENE_CONTINUITY = False
```

This runs once when Cell 6 executes. Contrast with the other toggles (`ENABLE_QUALITY_CHECKS`, `ENABLE_LLM_PROMPT_EXPANSION`, `ENABLE_VISION_ANALYSIS`) which are resolved inside `generate_pro()` at call time — those react to toggle changes without re-running upstream cells. Scene continuity is the only toggle with this cell-execution-time binding rather than call-time binding, because `USE_SCENE_CONTINUITY` is consumed by `run_storyboard()` which captures it as a default parameter value at definition time.
